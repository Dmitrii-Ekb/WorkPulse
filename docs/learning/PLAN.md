# План занятий WorkPulse

Обновлено: **2026-07-21**

## Формат работы

Работа строится короткими практическими итерациями:

1. Выбирается одна понятная задача.
2. Перед новым понятием даётся необходимое объяснение.
3. Ученик предлагает решение или прогнозирует результат.
4. Вносится минимальное изменение.
5. Выполняется подходящая проверка.
6. Ученик объясняет результат своими словами.
7. Наставник уточняет модель и исправляет неточности.
8. После законченного этапа предлагается название коммита.
9. Commit и push ученик выполняет самостоятельно.
10. После заметного этапа обновляется документация.

После того как закономерность понята, однотипные механические изменения можно выполнять сразу в нескольких местах и проверять общим прогоном.

Не задавать вопрос на знание конструкции, которая ещё не вводилась. Сначала кратко объяснить новое средство, затем дать применить его самостоятельно.

### 10. Выходной DTO и границы API

Статус: **завершён**

#### Проблема

Сущность `WorkTask` возвращалась напрямую. Новое внутреннее публичное свойство могло автоматически попасть в JSON.

#### Решение

Создан:

```text
WorkTaskResponseDto
```

Роли:

```text
CreateWorkTaskDto   — входной контракт
WorkTask            — сущность EF Core
WorkTaskResponseDto — выходной контракт
```

#### Реализация

Добавлен метод:

```csharp
public static WorkTaskResponseDto FromEntity(WorkTask task)
```

Он используется в:

- `GET /tasks/first`;
- `GET /tasks/{id}`;
- `POST /tasks`;
- `PATCH /tasks/{id}/complete`.

`GET /tasks` использует проекцию через `Select`.

`DELETE` DTO не требует, потому что возвращает `204 No Content`.

#### Изученный LINQ

```text
Where  — фильтрация
Select — преобразование
```

#### Проверки

Проверены:

- список задач;
- первая задача;
- задача по `Id`;
- создание;
- завершение;
- удаление;
- сохранение прежней формы JSON;
- одиночный объект вместо случайного массива.

#### Устное закрепление

Ученик объяснил:

- расшифровку DTO;
- роли входного DTO, сущности и выходного DTO;
- пользу стабильного контракта;
- когда свойство добавляется только в сущность;
- когда свойство нужно добавить в DTO ответа;
- базовую механику миграций.
