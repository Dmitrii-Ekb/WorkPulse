# Карта обучения WorkPulse

## Цель маршрута

Не просто собрать учебный API, а научиться самостоятельно проектировать, писать, отлаживать и развивать .NET-сервисы.

Конечная цель — уверенная база уровня middle-разработчика и проект, который не стыдно обсуждать на собеседовании.

Маршрут не привязан к жёстким срокам. Переход к следующей теме происходит после практического понимания предыдущей.

## Текущая веха

WorkPulse имеет Minimal API и постоянное хранение задач в SQLite через EF Core.

Работает вертикальная цепочка:

```text
HTTP-запрос
    ↓
входной DTO
    ↓
асинхронный endpoint и валидация
    ↓
сущность WorkTask
    ↓
DbContext
    ↓
асинхронная операция EF Core
    ↓
SQLite
    ↓
HTTP-ответ
```

Завершены улучшения:

- строка подключения вынесена в `appsettings.json`;
- обязательная настройка проверяется при запуске;
- `WorkTask` вынесен из `Program.cs` в отдельный файл;
- создан `CreateWorkTaskDto`;
- входной контракт создания отделён от сущности базы;
- клиент больше не управляет `Id` и начальным значением `IsCompleted`;
- весь текущий CRUD переведён на `ToListAsync`, `FindAsync`, `FirstOrDefaultAsync` и `SaveChangesAsync`;
- успешные и ошибочные HTTP-сценарии проверены после перехода на асинхронность.

Текущая контрольная точка — закрепить понимание потоков, пула потоков, `Task`, `async` и `await` через вопросы ученика.

Следующая запланированная тема — увидеть SQL, который EF Core формирует из LINQ и операций изменения. Она начнётся только после завершения вопросов по асинхронности.
