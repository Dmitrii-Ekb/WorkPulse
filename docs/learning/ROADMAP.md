# Карта обучения WorkPulse

## Цель маршрута

Не просто собрать учебный API, а научиться самостоятельно проектировать, писать, отлаживать и развивать .NET-сервисы. Конечная цель — уверенная база уровня middle-разработчика и проект, который не стыдно обсуждать на собеседовании.

Маршрут не привязан к жёстким срокам. Переход к следующей теме происходит после практического понимания предыдущей.

## Этап 1. База C# через реальный код

Статус: **в процессе**

Темы:

- переменные, типы и выражения;
- условия, циклы и методы;
- массивы и `List<T>`;
- классы, объекты, свойства и конструкторы;
- ссылочные и значимые типы;
- `null` и nullable-типы;
- исключения;
- делегаты, лямбды и базовый LINQ;
- строки, даты и перечисления;
- обобщения;
- `async`/`await`;
- управление памятью на практическом уровне.

Результат: ученик уверенно читает и пишет обычный C#-код и может объяснить, что происходит с данными и объектами.

## Этап 2. HTTP и ASP.NET Core

Статус: **начат**

Темы:

- устройство HTTP-запроса и ответа;
- методы GET, POST, PUT, PATCH, DELETE;
- маршруты и параметры;
- JSON, сериализация и десериализация;
- коды статуса;
- Minimal API;
- валидация входных данных;
- middleware;
- конфигурация и окружения;
- обработка ошибок;
- Swagger/OpenAPI;
- асинхронные endpoint-ы.

Результат: полноценный Web API с понятным контрактом.

## Этап 3. Базы данных и SQL

Статус: **следующий этап**

Темы:

- зачем нужна СУБД;
- таблица, строка, столбец и схема;
- первичные и внешние ключи;
- ограничения и уникальность;
- базовые SELECT, INSERT, UPDATE и DELETE;
- фильтрация, сортировка и JOIN;
- индексы;
- транзакции;
- SQLite для обучения;
- PostgreSQL или SQL Server позже.

Результат: данные WorkPulse сохраняются после перезапуска и корректно связаны между собой.

## Этап 4. Entity Framework Core

Темы:

- ORM и её компромиссы;
- `DbContext` и `DbSet`;
- connection string;
- миграции;
- отслеживание изменений;
- запросы LINQ к базе;
- асинхронные операции;
- связи сущностей;
- проблема N+1;
- границы транзакций.

Результат: API работает с реальной базой через EF Core, а ученик понимает, какой SQL примерно выполняется.

## Этап 5. ООП и организация кода

Темы вводятся после того, как `Program.cs` станет неудобным:

- методы и ответственность;
- инкапсуляция;
- интерфейсы;
- композиция и наследование;
- контроллеры или endpoint-модули;
- сервисы;
- репозитории только при реальной необходимости;
- dependency injection;
- DTO и отделение API-контракта от сущностей базы;
- базовые принципы SOLID на конкретных проблемах проекта.

Результат: код можно развивать без превращения проекта в один большой файл.

## Этап 6. Качество и тестирование

Темы:

- unit-тесты;
- integration-тесты API;
- тестовая база;
- Arrange, Act, Assert;
- мок-объекты только там, где они помогают;
- логирование;
- централизованная обработка ошибок;
- валидация;
- рефакторинг;
- анализаторы и форматирование.

Результат: изменения можно вносить уверенно и проверять автоматически.

## Этап 7. Пользователи и безопасность

Темы:

- регистрация и вход;
- хеширование паролей;
- authentication и authorization;
- JWT или cookie-аутентификация;
- роли и права;
- защита секретов;
- CORS;
- типовые уязвимости Web API;
- rate limiting.

Результат: задачи принадлежат конкретным пользователям и защищены правилами доступа.

## Этап 8. Production-навыки

Темы:

- конфигурация по окружениям;
- Docker;
- CI/CD;
- публикация приложения;
- облачная база данных;
- health checks;
- структурированные логи;
- метрики и трассировка;
- резервное копирование;
- миграции при развёртывании.

Результат: WorkPulse доступен не только на локальном компьютере.

## Этап 9. Темы уровня middle

Темы:

- архитектурные компромиссы;
- границы модулей;
- транзакции и конкурентные изменения;
- optimistic concurrency;
- кэширование;
- фоновые задачи;
- очереди сообщений;
- производительность;
- профилирование;
- надёжность;
- code review;
- декомпозиция требований;
- оценка задач;
- технические решения и их аргументация.

Результат: ученик не только пишет код, но и способен обосновывать решения и видеть риски.
