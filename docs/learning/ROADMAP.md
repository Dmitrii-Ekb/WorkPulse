# Карта обучения WorkPulse

Обновлено: **2026-07-21**

## Цель

Через последовательное развитие WorkPulse выйти на уверенный уровень Middle C#/.NET-разработчика.

Итоговый результат:

- уверенное владение C#;
- проектирование ASP.NET Core API;
- работа с EF Core и SQL;
- автоматическое тестирование;
- понимание архитектурных решений;
- безопасность;
- Docker и production-навыки;
- проект для содержательного обсуждения на собеседовании.

Маршрут адаптивный. Практическое понимание важнее жёсткого календаря.

## Текущая веха

WorkPulse сейчас содержит:

- .NET 9 Minimal API;
- асинхронный CRUD;
- SQLite;
- Entity Framework Core;
- две миграции;
- входной DTO;
- выходной DTO;
- контролируемый JSON-контракт;
- nullable-описание задачи;
- ручную практику SQL;
- проекцию через LINQ `Select`.

Текущая цепочка:

```text
HTTP-запрос
    ↓
CreateWorkTaskDto
    ↓
валидация
    ↓
WorkTask
    ↓
EF Core и SQLite
    ↓
WorkTaskResponseDto
    ↓
HTTP-ответ
```

Контрольная точка:

```text
асинхронный CRUD работает
        ↓
SQL прочитан и написан вручную
        ↓
схема изменена второй миграцией
        ↓
входной и выходной контракты разделены
        ↓
следующий шаг: интеграционный тест
```

## Этап 5. Контракты и организация кода

Статус: **в процессе**

Завершено:

- сущность вынесена из `Program.cs`;
- создан входной DTO;
- создан выходной DTO;
- выполнено явное преобразование;
- mapping одиночного объекта вынесен в `FromEntity`;
- список проецируется через `Select`;
- сущность больше не публикуется напрямую;
- AutoMapper не добавлен без необходимости.

Модель ответственности:

```text
CreateWorkTaskDto   — приём данных
WorkTask            — хранение и EF Core
WorkTaskResponseDto — публикация данных
```

Предстоит:

- DTO редактирования;
- методы и ответственность системно;
- размещение mapping-кода при росте проекта;
- сервисы;
- endpoint-модули или контроллеры;
- интерфейсы;
- композиция и наследование;
- SOLID на реальных проблемах;
- слои после появления достаточной сложности.

Результат: понятный код без преждевременной архитектуры.
